D1
Execution is fully deterministic given identical initial state and input sequence — no ambient state, no side-channel entropy.
determinism · pure-computation

State transitions form a monotone lattice — values may only advance, never regress. Enables wait-free reads across distributed nodes.
monotone · CRDT · wait-free
D3
Causal ordering enforced via vector clocks — no message may be processed before its causal predecessors are committed.
vector-clock · causal-consistency
D4
All operations are idempotent under a content-addressed deduplication key derived from the causal parent hash.
idempotency · content-hash · exactly-once
D5
The execution graph is a DAG — cycles are rejected at compile time via topological sort. Mutual causation modeled as shared parent.
DAG · topo-sort · compile-time

gossip · lamport · clock-sync

BFT · consensus · 2f+1

rollback · DAG-replay · ancestor-hash

fault-injection · reproducibility · audit

⚡ Enterprise K8s Deployment

One-click Kubernetes deployment — generates all manifests, configures mesh networking, provisions governance CRDs and billing webhooks.
Deployment Progress
1
Namespace + RBAC + ZeroTrust NetworkPolicy
Creates causalmesh namespace with ECDSA attestation RBAC and zero-trust network policies.
kubectl create namespace causalmesh kubectl apply -f rbac-ecdsa-zerotrust.yaml kubectl apply -f networkpolicy-mesh.yaml
2
Shard StatefulSets (8 shards × 3 replicas)
Deploys auto-sharded ledger nodes with consistent-hash routing across AZs.
kubectl apply -f shard-statefulset-8x.yaml kubectl rollout status statefulset/causalmesh-shard -n causalmesh
3
Inference Router + WebRTC + PeerJS Signaling
Multi-model router with PeerJS signaling server and STUN/TURN relay.
kubectl apply -f inference-router.yaml kubectl apply -f peerjs-signaling-svc.yaml kubectl apply -f stun-turn-config.yaml
4
Lamport + Vector Clock Daemon + Gossip
Initializes global Lamport clock daemons with per-node vector clock synchronization.
kubectl apply -f lamport-daemon-ds.yaml kubectl apply -f vector-clock-gossip.yaml
5
Governance CRDs + Audit Proofs + Rollback
Installs CRDs for policies, SHA-256 audit proofs, and causal rollback triggers.
kubectl apply -f governance-crd.yaml kubectl apply -f audit-proof-crd.yaml kubectl apply -f rollback-trigger-crd.yaml
6
Monetization + Metering Sidecar + Billing Webhook
Token accounting, per-inference metering, and enterprise billing webhook.
kubectl apply -f metering-sidecar.yaml kubectl apply -f billing-webhook.yaml # ✓ Mesh endpoint: https://mesh.causalmesh.svc:8443
